# 功能规格说明：内置子 Agent 支持

**特性分支**：`023-builtin-subagent`
**创建日期**：2025-12-08

## 用户场景与测试 *（必填）*

### 用户故事 1 - 基本子 Agent 访问（优先级：P1）

用户可以访问预配置的内置子 agent，而无需创建或配置自己的子 agent 文件。他们可以在安装系统后立即将任务委派给专门的 agent。

**为什么是这个优先级**：核心功能，无需用户配置即可提供即时价值并展示子 agent 系统的能力。

**独立测试**：可以通过使用内置子 agent 名称调用 Task 工具来全面测试，验证子 agent 成功执行并交付专门的任务完成。

**验收场景**：

1. **假设**全新安装且没有用户配置的子 agent，**当**用户使用 "Explore" 子 agent 类型调用 Task 工具时，**则**内置的 explore agent 成功执行并返回搜索结果
2. **假设**内置子 agent 可用，**当**用户调用 Task 工具但未提供有效的 subagent_type 时，**则**系统列出可用的子 agent，包括内置和用户配置的

### 用户故事 2 - 一致的子 Agent 接口（优先级：P2）

内置子 agent 与用户配置的子 agent 工作方式完全相同，在所有子 agent 类型中提供一致的体验。

**为什么是这个优先级**：确保架构一致性，防止用户对内置和自定义子 agent 产生混淆。

**独立测试**：可以通过验证内置子 agent 产生与用户配置的子 agent 相同的消息结构、工具限制和执行行为来测试。

**验收场景**：

1. **假设**一个内置子 agent 和一个用户子 agent，**当**两者执行类似任务时，**则**它们的输出格式、状态报告和生命周期管理是相同的
2. **假设**内置子 agent 被执行，**当**任务完成时，**则**它们创建与用户配置的子 agent 相同的 SubagentBlock 结构

## 需求 *（必填）*

### 功能需求

- **FR-001**：系统必须从内部源代码而非外部文件加载内置子 agent 配置
- **FR-002**：系统必须使内置子 agent 立即可用，无需用户配置
- **FR-003**：内置子 agent 必须与现有的 SubagentManager 和 Task 工具无缝集成
- **FR-004**：系统必须包含用于代码库探索任务的 "Explore" 内置子 agent
- **FR-005**：内置子 agent 必须支持与基于文件的子 agent 相同的配置选项（tools、model、systemPrompt）
- **FR-006**：系统必须处理内置、用户和项目子 agent 之间的优先级排序
- **FR-007**：内置子 agent 必须出现在 Task 工具描述中，以及在提供无效子 agent 类型时的错误消息中
- **FR-008**：系统必须允许内置子 agent 名称被更高优先级的用户配置覆盖
- **FR-009**：内置子 agent 必须支持工具过滤和模型指定，如同标准子 agent
- **FR-010**：系统必须优雅地处理缺失或无效的内置子 agent 定义

### 关键实体 *（如果功能涉及数据则包含）*

- **BuiltinSubagentDefinition**：表示硬编码的子 agent 配置，包含 name、description、systemPrompt、允许的工具和模型偏好
- **SubagentSource**：枚举，标识子 agent 来自内置定义、用户文件还是项目文件
- **SubagentPriority**：排序系统，确保内置子 agent 具有适当的优先级（通常低于项目/用户配置）
