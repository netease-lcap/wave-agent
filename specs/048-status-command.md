# 功能规格说明：Status 命令

**特性分支**：`048-status-command`
**创建日期**：2026-02-27

## 用户场景与测试 *（必填）*

### 用户故事 1 - 查看 Agent 状态（优先级：P1）

作为用户，我希望快速查看 agent 的当前配置和会话详情，以便验证我正在工作的环境。

**为什么是这个优先级**：这是请求的核心功能。它提供对 agent 状态的透明性，并有助于故障排除或验证设置。

**独立测试**：可以通过在 CLI 中输入 `/status` 并验证显示的信息（Version、Session ID、cwd、Wave base URL、Model）准确且格式正确来进行完整测试。

**验收场景**：

1. **假设** agent 处于交互状态，**当**用户输入 `/status` 时，**则** agent 显示包含版本、会话 ID、当前工作目录、Wave base URL 和当前模型的状态覆盖层，输入框被隐藏。
2. **假设**状态信息已显示，**当**用户按 `Esc` 时，**则**状态显示被关闭，输入框恢复，用户返回之前的交互状态。

---

### 边界情况

- **缺失信息**：如果某些元数据（如模型名称）尚未初始化或不可用，系统应显示占位符（例如"Unknown"或"Not set"），而不是崩溃或显示空白。
- **长路径**：如果当前工作目录路径非常长，显示应优雅地处理（例如通过换行或用省略号截断），以避免破坏 UI 布局。
- **快速调用**：如果用户快速多次输入 `/status`，系统应处理而不创建多个重叠的状态视图。

## 需求 *（必填）*

### 功能需求

- **FR-001**：系统必须在交互式 CLI 中识别 `/status` 命令。
- **FR-002**：系统必须显示 agent 软件的当前版本。
- **FR-003**：系统必须显示与当前执行关联的唯一会话 ID。
- **FR-004**：系统必须显示当前工作目录（cwd）的绝对路径。
- **FR-005**：系统必须显示为 agent 配置的 Wave base URL。
- **FR-006**：系统必须显示当前会话中活跃的 AI 模型名称。
- **FR-007**：系统必须显示提示"Esc to cancel"以告知用户如何关闭视图。
- **FR-008**：当按下 `Esc` 键时，系统必须关闭状态视图并返回之前的状态。
- **FR-009**：当状态视图活跃时，系统必须隐藏输入框。

### 关键实体

- **Session Metadata**：描述 agent 当前状态和配置的属性集合（Version、Session ID、CWD、Base URL、Model）。
