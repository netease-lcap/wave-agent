# 功能规格说明：清除命令

**特性分支**：`024-clear-command`
**创建日期**：2026-04-01

## 用户场景与测试 *（必填）*

### 用户故事 1 - 通过斜杠命令清除对话（优先级：P1）

作为 CLI 用户，我希望输入 `/clear` 来重置对话历史，以便在不重启应用的情况下开始全新会话。

**验收场景**：

1. **假设** 存在一个活跃对话且有消息，**当** 用户输入 `/clear`，**则** 对话历史被清除，生成新的会话 ID，并清除终端屏幕。
2. **假设** 正在处理中的 AI 响应，**当** 用户输入 `/clear`，**则** 正在进行的 AI 处理在清除对话前被中止。
3. **假设** 任务列表包含当前会话的项目，**当** `/clear` 执行后，**则** 任务列表与新的会话 ID 同步。

---

### 用户故事 2 - 通过 SDK 程序化清除（优先级：P1）

作为 SDK 消费者，我希望调用 `agent.clearMessages()` 来重置对话，以便我的应用程序可以程序化地开始新会话。

**验收场景**：

1. **假设** 存在一个活跃的代理会话且有消息，**当** 调用 `agent.clearMessages()`，**则** 对话历史被清除并生成新的会话 ID，效果与输入 `/clear` 相同。

---

### 边界情况

- **如果在没有对话的情况下调用 `/clear` 会发生什么？** 仍然会生成新的会话 ID；不会抛出错误。

## 需求 *（必填）*

### 功能需求

- **FR-001**：CLI 必须在 `AVAILABLE_COMMANDS` 中将 `clear` 注册为 CLI 内部命令，清除逻辑在 `Agent.clearMessages()` 中实现。
- **FR-002**：执行 `/clear` 必须中止任何正在进行的 AI 消息处理。
- **FR-003**：执行 `/clear` 必须清除对话历史并生成新的会话 ID。
- **FR-004**：执行 `/clear` 必须将任务列表与新的会话 ID 同步。
- **FR-005**：`Agent` 类必须暴露异步 `clearMessages()` 方法，其中直接包含清除逻辑（中止 AI、清除消息、同步任务）。
- **FR-006**：CLI 必须通过清除终端屏幕并重新挂载聊天界面来响应会话 ID 变更。
