# 功能规格说明：会话管理

**规格文件**：`specs/004-session-management.md`
**创建日期**：2026-01-21

## 用户场景与测试 *(必填)*

### 用户故事 1 - 基于项目的组织（优先级：P1）

作为处理多个项目的开发者，我希望我的代理会话按项目的工作目录组织，以便我能轻松找到和管理与当前工作相关的会话。

**优先级原因**：面向开发者的代理的核心组织需求。

**独立测试**：在不同目录中创建会话并验证它们存储在 `~/.wave/projects` 下的独立子目录中。

**验收场景**：

1. **假设** 有工作目录，**当** 创建会话时，**则** 它必须存储在以编码路径命名的子目录中。
2. **假设** 有较长或复杂的路径，**当** 编码时，**则** 它必须是文件系统安全的且限制为 200 个字符。

---

### 用户故事 2 - 高性能会话列表（优先级：P1）

作为拥有数百个历史会话的用户，我希望会话列表能快速加载，以便在启动或切换会话时不会体验到延迟。

**优先级原因**：随着会话数量增长，这对良好的用户体验至关重要。

**独立测试**：生成 100 个大会话文件并测量列出它们的时间；应该是近乎即时的，因为系统从单个 `sessions-index.json` 文件读取。

**验收场景**：

1. **假设** 有多个会话文件，**当** 列出会话时，**则** 系统必须从 `sessions-index.json` 读取以实现 O(1) 性能。
2. **假设** 索引缺失或损坏，**当** 列出会话时，**则** 系统必须回退到读取每个文件的最后一行并重建索引。

---

### 用户故事 3 - 子代理会话分离（优先级：P2）

作为使用子代理的开发者，我希望子代理会话被清楚地标识并与主代理会话分离，以避免污染主会话历史。

**优先级原因**：在处理复杂的多代理工作流时提高清晰度。

**独立测试**：启动子代理并验证其会话文件以 `subagent-` 为前缀。

**验收场景**：

1. **假设** 子代理已创建，**当** 其会话被保存时，**则** 文件名必须以 `subagent-` 开头。

---

### 边界情况

- **权限问题**：如果无法创建或写入会话目录，系统应优雅地失败并给出清晰的错误提示。
- **损坏的文件**：如果 JSONL 文件包含无效 JSON，在列出时应跳过，在加载时应视为不存在。
- **空会话**：没有消息的文件应使用文件的修改时间作为 `lastActiveAt` 时间戳。
- **路径编码冲突**：使用哈希后缀确保长路径即使被截断也具有唯一性。

## 需求 *(必填)*

### 功能需求

- **FR-001**：所有会话数据必须存储在 `~/.wave/projects` 中。
- **FR-002**：会话必须根据项目工作目录路径的编码版本分组到子目录中。
- **FR-003**：工作目录路径必须编码为文件系统安全的格式，解析符号链接并处理特殊字符。
- **FR-004**：编码的目录名必须限制为 200 个字符，对更长的路径使用哈希后缀。
- **FR-005**：会话必须以 JSONL（JSON Lines）格式存储，每行一个 JSON 对象表示一条消息。
- **FR-006**：主会话文件必须使用 UUID 命名。链中的第一个会话 ID 必须被指定为 `rootSessionId` 并持久化在 `SessionIndex` 中。
- **FR-007**：`rootSessionId` 必须在消息压缩和会话恢复之间保留，以为会话范围内的资源（如计划文件和任务列表）提供稳定的标识符。
- **FR-008**：子代理会话文件必须以 `subagent-` 为前缀。
- **FR-008**：每条消息行必须包含 ISO 8601 格式的 `timestamp` 字段。
- **FR-009**：会话元数据必须从文件名、目录结构和最后一条消息行中派生。
- **FR-010**：会话列表必须优化为使用 `sessions-index.json` 以实现 O(1) 性能，最小化 I/O。
- **FR-011**：超过 14 天的会话必须自动清理。
- **FR-012**：清理期间必须移除空的项目目录。
- **FR-013**：每个项目目录必须维护一个 `sessions-index.json` 文件用于 O(1) 会话列表。
- **FR-014**：会话索引必须缓存 `firstMessage` 内容以实现即时 UI 显示。
- **FR-015**：如果会话索引缺失或损坏，系统必须能够从 `.jsonl` 文件重建。
- **FR-016**：`firstMessage` 字段必须跳过元消息（`isMeta: true`）并捕获第一个非元消息内容用于会话列表显示。

### 关键实体

- **Session**：用户与代理之间对话的记录。
    - `id`：当前会话的 UUID。
    - `rootSessionId`：链中第一个会话的 UUID（跨压缩持久化）。
    - `type`：'main' 或 'subagent'。
    - `workdir`：与会话关联的项目目录。
    - `lastActiveAt`：最后一条消息的时间戳。
    - `messages`：消息对象列表。
    - `firstMessage`：（可选）会话中第一条消息的内容，用于显示目的。
- **SessionIndex**：
    - `sessions`：会话 ID 到元数据的映射。
    - `lastUpdated`：最后一次索引更新的时间戳。
- **项目目录**：属于特定路径的会话容器。
    - `path`：原始文件系统路径。
    - `encodedPath`：用于目录的文件系统安全名称。

## 假设

- 用户的家目录可访问且可写。
- JSONL 是一种高效的格式，可以在不重写整个文件的情况下追加消息。
