# 功能规格说明：/loop 斜杠命令

**特性分支**：`051-loop-slash-command`
**创建日期**：2026-03-24

## 用户场景与测试 *（必填）*

### 用户故事 1 - 调度循环任务（优先级：P1）

作为用户，我希望调度命令或提示以固定间隔运行，以便监控长时间运行的进程或获取定期更新而无需手动干预。

**为什么是这个优先级**：这是 `/loop` 命令的核心功能。

**独立测试**：可以通过运行 `/loop 5m /echo "hello"` 或 `/loop 5m check the build` 并验证任务已调度并立即运行来进行测试。

**验收场景**：

1. **假设** agent 处于空闲状态，**当**我输入 `/loop 5m /echo "ping"` 时，**则**系统应该调度一个 cron 任务，确认调度并附带任务 ID，并立即输出"ping"。
2. **假设** agent 处于空闲状态，**当**我输入 `/loop check the build every 2h` 时，**则**系统应该调度一个 cron 任务，确认调度并立即执行"check the build"。

---

### 用户故事 2 - 使用默认间隔调度（优先级：P2）

作为用户，我希望快速循环执行命令或提示而无需指定时间，以便使用合理的默认频率。

**为什么是这个优先级**：通过减少常见情况的输入来改善用户体验。

**独立测试**：可以通过运行 `/loop /check-build` 或 `/loop check the build` 并验证它默认为合理间隔（例如 10 分钟）来进行测试。

**验收场景**：

1. **假设** agent 处于空闲状态，**当**我输入 `/loop check the build` 时，**则**系统应该为默认间隔调度任务并确认频率。

---

### 边界情况

- **空提示**：如果我输入 `/loop 5m` 不带命令，系统应显示使用说明并不调度任何内容。
- **惊群预防**：如果我要求"hourly"，系统应选择随机分钟（例如 7）而不是 0，以避免全局同步问题。
- **多会话安全**：当多个 Wave 会话在同一项目中运行时，只有一个会话触发持久任务（通过调度器租约锁）。非所有者会话回退为触发仅会话任务。
- **锁接管**：如果拥有锁的会话退出，另一个会话在 5 秒内接管锁并恢复触发持久任务。

## 需求 *（必填）*

### 功能需求

- **FR-001**：系统必须支持 `/loop [interval] <prompt>`，其中间隔和提示由 AI 根据 skill 定义解析。
- **FR-006**：系统必须使用 `CronCreate` 工具将任务调度为循环任务。
- **FR-007**：系统必须在成功调度后立即执行解析后的提示一次。
- **FR-008**：系统必须提供确认消息，包含：
    - 人类可读的频率。
    - cron 表达式。
    - Job ID。
    - 循环任务在 7 天后自动过期的提醒。
    - 如何使用自然语言取消任务的说明（例如"stop loop [ID]"）。
- **FR-009**：如果结果提示为空，系统必须显示使用说明。
- **FR-010**：系统必须支持 `CronCreate` 上的 `durable` 参数，将任务持久化到 `.wave/scheduled_tasks.json`。`durable: false`（默认）的任务仅限会话。
- **FR-011**：持久任务必须仅从持有调度器租约锁的会话触发（防止跨多个会话的双重触发）。
- **FR-012**：当拥有锁的会话退出时，另一个会话必须接管锁并恢复触发持久任务。
- **FR-013**：`CronList` 必须用其作用域（`durable` 或 `session-only`）标注每个任务。
- **FR-014**：`CronDelete` 必须从内存和磁盘存储中移除持久任务。

### 关键实体

- **Loop Job**：调度的循环任务。
    - **Job ID**：用于取消的唯一标识符。
    - **Cron Expression**：cron 格式的调度计划。
    - **Prompt**：要执行的命令或文本。
    - **Cadence**：频率的人类可读描述。
    - **Expiration**：任务自动删除的日期/时间（创建后 7 天）。
    - **Scope**：`session-only`（内存中，退出时消亡）或 `durable`（文件支持，重启后保留）。
    - **Scheduler Lock**：持久任务需要租约锁以防止跨会话的双重触发。
