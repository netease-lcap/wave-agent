# 功能规格说明：帮助命令

**特性分支**：`027-help-command`
**创建日期**：2026-03-25

## 用户场景与测试 *（必填）*

### 用户故事 1 - 访问帮助（优先级：P1）

作为用户，我希望能够通过输入 `/help` 查看可用命令和快捷键的列表，以便我可以有效地学习如何使用 agent。

**为什么是这个优先级**：这是核心功能。它为所有其他功能提供可发现性。

**独立测试**：在聊天输入中输入 `/help` 并验证帮助视图出现。

**验收场景**：

1. **假设** agent 正在运行，**当**用户输入 `/help` 并按下 Enter 时，**则**帮助视图被显示。
2. **假设**帮助视图已打开，**当**用户按下 `Esc` 时，**则**帮助视图关闭，用户返回聊天输入。

---

### 用户故事 2 - 导航帮助标签页（优先级：P2）

作为用户，我希望使用 `Tab` 键在帮助的不同类别（通用、命令、自定义命令）之间切换，以便我可以快速找到所需的信息。

**为什么是这个优先级**：改善帮助信息的组织性和可用性。

**独立测试**：打开帮助，按下 `Tab`，验证活动标签页变化。

**验收场景**：

1. **假设**帮助视图在"通用"标签页打开，**当**用户按下 `Tab` 时，**则**"命令"标签页变为活动。
2. **假设**"命令"标签页活动，**当**用户按下 `Tab` 时，**则**"自定义命令"标签页变为活动（如果有自定义命令存在），否则循环回到"通用"。

---

### 用户故事 3 - 浏览命令（优先级：P2）

作为用户，我希望滚动查看命令列表并看到它们的描述，以便我可以理解每个命令的作用。

**为什么是这个优先级**：提供每个命令的详细信息。

**独立测试**：打开帮助，切换到"命令"标签页，使用方向键导航，验证描述被显示。

**验收场景**：

1. **假设**"命令"标签页活动，**当**用户使用 `Up` 和 `Down` 方向键时，**则**选中的命令改变。
2. **假设**一个命令被选中，**则**其描述显示在命令名称下方。
3. **假设**命令数量超过视图可显示的范围，**当**用户向下滚动时，**则**命令列表滚动以显示更多项。

---

## 需求 *（必填）*

### 功能需求

- **FR-001**：系统必须提供 `/help` 内置命令。
- **FR-002**：系统必须在调用 `/help` 时显示交互式帮助视图。
- **FR-003**：帮助视图必须包含显示快捷键的"通用"部分。
- **FR-004**：帮助视图必须包含显示所有内置斜杠命令的"命令"部分。
- **FR-005**：帮助视图必须包含"自定义命令"部分（如果有任何插件或自定义命令已注册）。
- **FR-006**：系统必须允许使用 `Tab` 键在帮助部分之间切换。
- **FR-007**：系统必须允许使用 `Up` 和 `Down` 方向键导航命令列表。
- **FR-008**：系统必须显示当前选中命令的描述。
- **FR-009**：系统必须允许使用 `Esc` 键关闭帮助视图。
- **FR-010**：帮助视图必须使用 Ink 组件实现，以保持与 CLI 的一致性。

### 关键实体 *（如果功能涉及数据则包含）*

- **SlashCommand**：表示具有 ID、名称、描述和处理程序的命令。
- **HelpItem**：用于通用帮助的键-描述对（如 "@" -> "引用文件"）。
- **HelpView**：负责渲染帮助信息的 UI 组件。
